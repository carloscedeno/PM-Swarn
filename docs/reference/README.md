# Reference (context sharding)

Modular context docs — load only what the current task needs.

Add per-topic files, e.g.:

- `auth.md` — Authentication flow, token management, permissions
- `workflow.md` — Process and workflow decisions
- `api_endpoints.md` — Endpoints with request/response examples
- `db_schema.md` — Tables, relationships, indexes
- `env_variables.md` — Environment variables and purpose
